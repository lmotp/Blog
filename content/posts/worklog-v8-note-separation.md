## 작업 배경

JavaScript 메모가 길어지면서, V8 최적화 이야기는 별도의 메모로 분리하는 편이 읽기에도, 관리하기에도 더 낫다고 판단했습니다.

메모는 가볍고 빠르게 훑을 수 있어야 하고, V8처럼 설명이 긴 주제는 별도 글로 떨어져 있는 편이 더 자연스럽습니다.

## 정리한 내용

- `content/notes/javascript.md`에서 V8 섹션을 분리했습니다.
- `content/notes/v8-optimization.md`를 별도 메모로 추가했습니다.
- JavaScript 메모는 Promise, iterator, generator 같은 메모 중심 내용으로 정리했습니다.
- 긴 기술 설명은 하나의 독립된 노트로 관리하도록 바꿨습니다.

## 작업하면서 느낀 점

내용을 분리하면 글이 짧아지는 대신, 읽는 흐름이 또렷해집니다.

특히 메모와 작업일지를 섞지 않으면, 각 페이지가 맡아야 하는 역할이 더 분명해집니다.
