## 작업 배경

블로그 작업을 계속하다 보면 화면 구조나 문구만큼이나, 코드가 지켜야 할 기준도 같이 정리해두는 편이 좋습니다.

특히 이 저장소는 접근성과 타입 안정성을 중요하게 보므로, 규칙을 문서와 코드에 같이 남겨두는 쪽이 유지보수에 유리합니다.

## 정리한 내용

- `AGENTS.md`에 접근성 우선 규칙을 추가했습니다.
- TypeScript는 `strict` 기준으로 유지하도록 정리했습니다.
- `any`와 불필요한 `as` 타입 단언을 지양하도록 맞췄습니다.
- UI 로직과 비즈니스 로직을 분리하는 방향을 분명히 했습니다.

## 작업하면서 느낀 점

규칙은 한 번 적어두면 끝나는 문서가 아니라, 실제 코드를 고를 때마다 기준이 되는 메모에 가깝습니다.

그래서 작업일지에는 결과만 적기보다, 어떤 기준으로 코드를 선택했는지도 같이 남겨두는 편이 좋습니다.
